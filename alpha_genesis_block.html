Identity Blockchain: Genesis Block

ALPHA: Sept 10 2015 00:00:00 UTC

What are you talking about?

This page describes the start of the Identity Blockchain: a tool for validating secure digital identity documents, processing electronic signatures, and recording transactions. Since this is a new blockchain, it begins with a Genesis Block, shown below. All subsequent data entries in the blockchain must reference this genesis block through a chain of strong cryptographic proofs. Any attempt at manipulation of blockchain data will break the chain of proofs, making it effectively impossible to alter records without detection.

The genesis block is structured as any other typical block, with the exception that it references no previous block and includes no transactions. All values are represented in hexadecimal, i.e. base 16, with "numbers" running from zero through the letter f (0123456789abcdef). Hexadecimal is a convenient way to represent the binary data of computers in a form more easily readable to humans.

Numbers are stored in "big-endian" order where the "big end" (most significant value in the sequence) is stored first. The term "big-endian" derives from Jonathan Swift's Gulliver's Travels in which the Big Endians were a political faction who broke their eggs at the large end.

Show me the Genesis Block!

Genesis Block - The first block in the Identity Blockchain!

MagicNo: The "Magic Number" is an unique identifier for the blockchain, and remains constant for all subsequent blocks.

Timestamp: The genesis block timestamp is September 10, 2015 at midnight UTC. Time is represented in a UNIX format, which counts up in seconds from January 1, 1970. The timestamp is encoded as double (primary data type).

Version: The genesis block shows a version of ffff ffff (the highest hexadecimal value possible) to indicate that it is an alpha version, still in development.

HashPrevBlock: The genesis block is the first blockchain entry, so there is no "previous block" to reference. Therefore, its HashPrevBlock value is 0 (zero). The genesis block's own block identifier (described below) then serves as the HashPrevBlock value for the second block in the blockchain.

HashMerkleRoot: The HashMerkleRoot is computed from transaction data. It is named for Ralph Merkle, who received a patent in 1979 for the efficient and secure verification of large data structures using a "hash tree". The genesis block contains no transactions, so its HashMerkleRoot value is 0 (zero).

How is the block identifier derived?

The colored numbers from the genesis block are used to compute a "block identifier" signature shown in the black box above. The computation is a one-way function known as a cryptographic hash. This allows one to easily verify that some input data maps to a given hash value, but if the input data is unknown, it is deliberately difficult to reconstruct it (or equivalent alternatives) by knowing the stored hash value.

The particular function used here is a 256 bit "Secure Hash Algorithm" known as SHA 256, which was patented by the US National Security Agency and subsequently released under a royalty-free license. This function has been in the public domain since 2001, and is generally considered by cryptography experts to be extremely secure.
